Use case 1: Fixing the dark mode toggle across pages

This was one of the most useful but also most frustrating parts of the project.

Prompt used:
“My dark mode toggle works on one page but not on the others. How can I make the theme stay consistent across all pages?”

AI output summary:
ChatGPT suggested using JavaScript with localStorage to save the theme and applying a shared class such as dark-page to the body element on every page. It also suggested using the same toggle ID and reusing the same script file across the site.

Changes I made:
I did not use the response exactly as it was given. I adapted the code to match my own page structure, my existing CSS classes and my file setup. I connected the toggle through main.js, reused the same theme key in local storage, and added dark mode CSS for cards, text, navigation and form elements. I also had to correct some page-specific styling because simply adding the toggle logic did not make the whole design switch properly.

What limitation I noticed:
The first answers looked correct, but they were not complete for my project. Even after adding the suggested code, the toggle either did not work on some pages, did not carry over correctly, or changed only part of the page. In some cases the container styling remained light while the background changed dark, which made the design look broken.

Critical evaluation:
One part of the AI output was subtly incorrect because it assumed that applying a single dark-page class would automatically update all components. While this sounded correct, it did not account for my existing page-specific CSS, which caused inconsistent styling across pages. I identified this issue when some elements remained in light mode even though the background changed, which required manual CSS adjustments.

Use case 2: Troubleshooting the live preview card

This was an example where AI helped me think through the problem, but not every suggestion solved it immediately.

Prompt used:
“Why are the changes I make not appearing properly in the live preview card, and what should I check?”

AI output summary:
ChatGPT suggested checking whether the correct selectors were being targeted, whether the JavaScript was updating the right HTML elements, and whether the preview area needed better structure or event handling. It also suggested that I review my CSS and content containers to make sure the updates were visible.

Changes I made:
I used the response as a guide, but I had to inspect my own HTML, CSS and JavaScript closely. I adjusted the layout structure, checked whether the correct elements were being selected, and refined the preview section so that updates were actually visible on screen instead of only changing the code behind the scenes.

What limitation I noticed:
The AI suggestions were useful for troubleshooting, but they were broad. They did not immediately identify the exact problem in my page. I still had to test each part myself and compare the visual output with the expected behaviour.

Critical evaluation:
A subtle limitation in the AI response was that it provided general debugging steps rather than identifying the exact issue. The suggestions appeared correct, but they did not directly resolve the problem because they were not specific to my DOM structure. I realised this when applying the suggestions did not fix the preview, which required deeper inspection of my own code.

Use case 3: Making the quiz shuffle properly on page load

This was useful because the quiz needed to feel interactive rather than static.

Prompt used:
“My quiz questions are staying in the same order even though I want them to shuffle each time the page loads. How can I fix that while still using my JSON questions?”

AI output summary:
ChatGPT suggested shuffling the array before rendering the questions, rather than changing the original JSON file itself. It explained that the JSON should remain as the source data, while the displayed question order should be randomised in JavaScript at runtime.

Changes I made:
I kept the original question structure in JSON, but adapted the rendering logic so the displayed order changed on load. I had to make sure the shuffled version still matched the rest of the quiz logic and did not break validation or answer handling.

What limitation I noticed:
Although the AI explained the concept clearly, the code still needed adjustment to fit my actual quiz setup. The first version did not fully align with the structure I was already using.

Critical evaluation:
The AI response was subtly incorrect in that the initial code example did not fully match my existing quiz structure, even though the logic was valid. It appeared usable at first, but caused issues with validation when implemented directly. I detected this when the quiz behaviour broke after integration, which required modifying the code to fit my setup.

Reliability, risks and ethical considerations

Across this project, ChatGPT was useful for debugging, planning and suggesting implementation approaches, especially when I was stuck on dark mode, quiz behaviour and interactive components. However, the responses were not always directly usable. In multiple cases, the suggestions sounded correct but still required adaptation before they worked properly in my own code.

A major risk was assuming that because the answer looked professional, it must also be fully correct. My experience showed that AI can be helpful without being complete. The most common issue was not an obviously wrong answer, but a partially correct one that still caused layout issues, inconsistent behaviour or missing styling.

There were also ethical considerations. I needed to acknowledge AI assistance clearly, avoid copying outputs without understanding them, and make sure the final work still reflected my own judgement and testing. For that reason, I treated ChatGPT as a support tool rather than a substitute for learning or problem-solving.

Overall, the most important lesson was that AI improved my efficiency, but responsibility for correctness stayed with me. The final quality of the website depended not on the generated code alone, but on my ability to test, evaluate, modify and justify the decisions I made.
